Retrieval-Augmented Generation, or RAG, is a pattern that improves language model responses by retrieving relevant external context before generating an answer. Instead of relying only on the model's internal knowledge, the system searches a document collection and injects the most relevant text into the prompt.

RAG systems typically include document ingestion, text preprocessing, chunking, embedding generation, vector storage, retrieval, and final answer generation. Chunking is important because large documents must be broken into smaller units before embeddings are created.

When chunking is done well, retrieval quality improves because each chunk contains focused information that can be matched more precisely to a user query.
